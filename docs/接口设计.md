# 接口设计

## 1. 接口规范
- 基础前缀：`/api`
- 返回格式：统一使用 `code`、`message`、`data`
- 认证方式：`Authorization: Bearer <token>`
- 数据格式：`application/json`

统一返回示例：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

## 2. 认证接口

### 2.1 用户登录
- `POST /api/auth/user/login`

请求参数：

```json
{
  "username": "user01",
  "password": "123456"
}
```

返回参数：

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "jwt-token",
    "tokenPrefix": "Bearer",
    "expiresIn": 86400,
    "role": "USER",
    "displayName": "普通用户"
  }
}
```

### 2.2 管理员登录
- `POST /api/auth/admin/login`

### 2.3 获取当前登录信息
- `GET /api/auth/profile`

## 3. 用户端初始化接口

### 3.1 获取用户端模块列表
- `GET /api/user/modules`

返回示例：

```json
{
  "code": 200,
  "message": "查询成功",
  "data": [
    {
      "code": "user-profile",
      "name": "个人中心",
      "path": "/user/profile",
      "description": "查看个人资料与账号信息"
    }
  ]
}
```

## 4. 管理端初始化接口

### 4.1 获取管理端模块列表
- `GET /api/admin/modules`

## 5. 第二阶段预留接口

### 5.1 用户管理
- `GET /api/admin/users`
- `POST /api/admin/users`
- `PUT /api/admin/users/{id}`
- `DELETE /api/admin/users/{id}`

### 5.2 区域管理
- `GET /api/admin/regions`
- `POST /api/admin/regions`
- `PUT /api/admin/regions/{id}`
- `DELETE /api/admin/regions/{id}`

### 5.3 车辆信息管理
- `GET /api/admin/cars`
- `POST /api/admin/cars`
- `PUT /api/admin/cars/{id}`
- `DELETE /api/admin/cars/{id}`

### 5.4 订单中心
- `GET /api/admin/orders`
- `GET /api/user/orders`

### 5.5 归还管理
- `POST /api/user/returns`
- `GET /api/admin/returns`
