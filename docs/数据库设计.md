# 数据库设计

## 1. 设计原则
- 以“区域、车辆、订单、归还”为主业务链设计
- 用户端与管理端账号分表，便于后续权限独立扩展
- 第一阶段优先保证核心实体与主流程数据完整
- 删除采用逻辑删除字段 `deleted`

## 2. 表清单

| 表名 | 说明 |
| --- | --- |
| `sys_user` | 用户表 |
| `sys_admin` | 管理员表 |
| `region` | 区域表 |
| `car_info` | 车辆信息表 |
| `car_deploy_record` | 车辆投放记录表 |
| `car_recovery_record` | 车辆回收记录表 |
| `rental_order` | 租赁订单表 |
| `return_record` | 归还记录表 |

## 3. 核心表设计说明

### 3.1 sys_user
存储用户基础信息、账号信息和状态信息，支撑用户端个人中心与订单归属。

关键字段：
- `username`：登录账号
- `password`：登录密码
- `real_name`：真实姓名
- `phone`：手机号
- `status`：账号状态

### 3.2 sys_admin
存储管理员账号和角色信息，支撑管理端登录与管理员管理。

关键字段：
- `username`：管理员账号
- `password`：密码
- `role_name`：角色名称，如 `SUPER_ADMIN`、`ADMIN`
- `status`：启停状态

### 3.3 region
用于管理车辆投放区域和用户可取还车地点。

关键字段：
- `region_name`：区域名称
- `region_code`：区域编码
- `address`：详细地址
- `longitude` / `latitude`：经纬度
- `status`：区域状态

### 3.4 car_info
车辆主档案表，管理车辆品牌、型号、租金和当前状态。

关键字段：
- `car_code`：车辆编号
- `license_plate`：车牌号
- `daily_rent`：日租金
- `deposit`：押金
- `current_region_id`：当前所属区域
- `status`：车辆状态

建议状态：
- `IDLE`：空闲
- `DEPLOYED`：已投放
- `RENTING`：租赁中
- `RETURN_PENDING`：待归还确认
- `RECOVERED`：已回收
- `MAINTENANCE`：维修中

### 3.5 car_deploy_record
记录车辆投放动作，便于审计车辆流转。

关键字段：
- `car_id`：车辆 ID
- `region_id`：投放区域
- `operator_admin_id`：操作管理员
- `deploy_time`：投放时间

### 3.6 car_recovery_record
记录车辆回收动作与回收原因。

关键字段：
- `car_id`：车辆 ID
- `region_id`：回收前区域
- `recovery_reason`：回收原因
- `operator_admin_id`：操作管理员

### 3.7 rental_order
订单主表，支撑用户下单、订单中心和租赁状态流转。

关键字段：
- `order_no`：订单号
- `user_id`：下单用户
- `car_id`：租赁车辆
- `pickup_region_id`：取车区域
- `return_region_id`：还车区域
- `order_status`：订单状态
- `payment_status`：支付状态

建议状态：
- `CREATED`
- `PAID`
- `USING`
- `RETURNING`
- `COMPLETED`
- `CANCELLED`

### 3.8 return_record
记录归还时的车辆检查结果和费用结算信息。

关键字段：
- `rental_order_id`：订单 ID
- `vehicle_condition`：车辆状态描述
- `damage_cost`：损坏费用
- `late_fee`：逾期费用
- `final_amount`：最终结算金额
- `processed_by`：处理管理员

## 4. 关系说明
- 一个区域可关联多辆车
- 一辆车可对应多次投放记录和回收记录
- 一个用户可拥有多个订单
- 一个订单对应一条归还记录
