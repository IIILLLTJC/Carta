# 需求说明

## 1. 项目背景
车辆租赁管理系统用于统一管理车辆投放、租赁下单、订单流转、车辆归还与后台运营管理，满足用户端自助租车和管理端运营管理的双端业务需求。

## 2. 功能范围

### 2.1 用户端模块
- 个人中心：查看和维护个人资料、联系方式、账号信息
- 车辆投放地点：查看可租区域、车辆投放点、车辆基础信息
- 订单管理：创建订单、查看订单、取消订单、查看租赁状态
- 归还管理：提交归还、查看归还记录、查看费用结算信息

### 2.2 管理端模块
- 个人中心：管理员资料维护、密码修改
- 用户管理：用户列表、状态管理、用户信息维护
- 区域管理：投放区域增删改查、位置与状态维护
- 车辆信息管理：车辆档案、状态、租金、押金、所在区域维护
- 车辆投放管理：记录投放操作、投放区域、操作人、时间
- 车辆回收管理：记录回收操作、原因、处理状态
- 订单中心：订单检索、状态跟踪、异常订单处理
- 归还管理：归还记录查看、归还费用确认、车辆状态更新
- 管理员管理：管理员新增、编辑、启停、角色区分

## 3. 角色定义
- 普通用户：租车、下单、查看车辆、发起归还
- 管理员：运营管理、车辆与区域管理、订单与归还处理
- 超级管理员：管理员管理与全局权限控制

## 4. 核心业务主线
1. 管理员创建区域并维护车辆基础信息
2. 管理员将车辆投放到指定区域
3. 用户浏览可租车辆并创建订单
4. 订单进入租赁中状态后，车辆状态变为租赁中
5. 用户归还车辆，系统记录归还信息并进行费用结算
6. 管理端根据归还结果决定车辆继续投放或回收处理

## 5. 第一阶段范围
本阶段仅完成基础架构和原型骨架，不一次性写完整业务，范围包括：

- 项目目录初始化
- 文档初始化
- 数据库设计初稿
- 后端基础工程搭建
- 前端基础工程搭建
- 登录页与双端布局骨架

## 6. 非功能要求
- 前后端分离
- 接口统一返回结构
- JWT 鉴权基础能力
- 支持跨域
- 代码结构清晰，便于后续扩展
